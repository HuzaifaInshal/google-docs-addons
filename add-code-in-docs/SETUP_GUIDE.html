What's in the package

You received 3 files. Each must be added to your Google Apps Script project exactly as named below.

⚙️

Code.gs

The backend brain. Handles all document manipulation — tokenizes your code, applies syntax colors, and inserts styled paragraphs into Google Docs. All logic lives here.

🎨

Sidebar.html

The sidebar UI — the dark-themed panel that opens inside Google Docs. Contains all dropdowns, toggles, the code editor, and the Insert button.

📋

appsscript.json

The manifest file. Declares required permissions (access to Docs + UI) and registers the add-on menu. This must be enabled in Apps Script settings to be editable.

🚀

Setup — 5 simple steps

ℹ️ You only do this setup once. After that, the add-on is available in every new Google Doc automatically.
1

Open Google Apps Script

Go to script.google.com and click + New project in the top-left. Name it "CodeBlocks Pro".

URL https://script.google.com
2

Enable the manifest file

In the Apps Script editor, click the gear icon ⚙️ (Project Settings) in the left sidebar. Check the box that says "Show 'appsscript.json' manifest file in editor". Then go back to the editor (the </> icon).

3

Add all 3 files

You'll see a default Code.gs file already there. Here's what to do for each file:


📄 Code.gs — Click the existing Code.gs tab. Select all content and delete it. Paste the full contents of your downloaded Code.gs file.


🌐 Sidebar.html — Click the + button next to "Files" in the left sidebar → choose HTML → name it exactly Sidebar (no extension needed — Apps Script adds .html automatically). Paste the contents of your Sidebar.html file.


📋 appsscript.json — Click the appsscript.json tab (visible after Step 2). Select all content and replace it entirely with the contents of your downloaded appsscript.json file.

⚠️ File names are case-sensitive. The HTML file must be named exactly Sidebar (capital S) — or the sidebar won't open.
4

Save & run once to authorize

Press Ctrl+S (or Cmd+S on Mac) to save. Then click the ▶ Run button in the toolbar (it will ask you to pick a function — choose onOpen).

Google will show an authorization popup. Click "Review Permissions" → choose your Google account → click "Advanced" → "Go to CodeBlocks Pro (unsafe)" → "Allow".

⚠️ The "unsafe" warning is normal for personal scripts — it just means the script hasn't been verified by Google. Since you wrote it yourself, it's completely safe to allow.
5

Open a Google Doc & test it

Open any Google Doc (or create a new one). You'll now see an Extensions menu in the top bar. Click it and look for CodeBlocks Pro → Open CodeBlocks Panel. The sidebar will open on the right side. You're ready! 🎉

✅ If you don't see the menu immediately, refresh the Google Doc page once.

How to use it

1

Open the panel

In Google Docs, go to Extensions → CodeBlocks Pro → Open CodeBlocks Panel. The sidebar appears on the right.

2

Place your cursor in the doc

Click somewhere in your document where you want the code block to appear. The block will be inserted right at that position (or at the end of the doc if no cursor position is found).

3

Choose language & theme

Use the two dropdowns to select your programming language and color theme. You can also click the colored dots below the dropdowns to quickly switch themes. The language badge in the code editor updates live.

4

Configure options

Toggle Line numbers on/off and show/hide the language header bar. Adjust font size with the +/− buttons (default is 11pt — good for most docs).

5

Paste your code & insert

Paste or type code in the editor area. Press Tab for indentation. Click the glowing "Insert Code Block" button. Done — your code appears in the doc within seconds!

💡 You can insert multiple code blocks in one session. Each block is independent — just change the language/theme/code and hit Insert again for the next one.

FAQ

Why does it say "Go to unsafe app" during authorization? ▾
This is completely normal for personal Google Apps Scripts. Google shows this warning for any script that hasn't gone through their official verification process (which costs money and is only for published marketplace add-ons). Since you're running your own code, it's entirely safe to proceed.
The "Extensions" menu doesn't show CodeBlocks Pro. What do I do? ▾
First, hard-refresh the Google Doc page (Ctrl+Shift+R or Cmd+Shift+R). If it still doesn't appear, go back to script.google.com, open your project, and manually run the onOpen function using the ▶ Run button (make sure to select "onOpen" from the function dropdown first).
Can I edit or delete a code block after inserting it? ▾
Yes! Code blocks are just styled paragraphs in your Google Doc. You can select them and delete them normally. Editing the code text directly will lose the syntax colors (since they're applied during insertion), so it's best to delete and re-insert if you need to change the code.
Does this work on Google Docs mobile? ▾
The add-on sidebar only works in the Google Docs web interface on desktop. However, code blocks you've already inserted will display correctly on mobile — the formatting (colors, fonts, backgrounds) is baked into the document.
Will the code block look right when I export to PDF? ▾
Yes! Since the colors and fonts are applied as native Google Docs text formatting, they export to PDF correctly. Background colors on paragraphs also export well. It's recommended to use File → Download → PDF for the best result.
How large can my code block be? ▾
There's no hard limit, but Google Docs may slow down for very large blocks (500+ lines). For very long code snippets, consider splitting into multiple smaller blocks. Insertion time increases proportionally with line count — a 50-line block typically inserts in about 3–6 seconds.
Can I share this add-on with my team? ▾
Yes — share the 3 files with teammates and have them follow the same setup steps. Each person needs their own Apps Script project. Alternatively, if you want centralized deployment, you can publish it as a private Google Workspace Marketplace add-on for your organization.
